Скрипт предназначен для автоматизированного поиска и замены текста в PDF-документах (например, дат или штампов) с сохранением исходного форматирования. Он работает на базе библиотеки PyMuPDF (fitz).

Алгоритм обработки:

Поиск: Скрипт проходит по всем страницам документа и ищет точные совпадения строк из списка OLD_TEXTS.
Анализ стиля: Для каждого найденного фрагмента определяется его точное положение (bbox), размер шрифта и цвет RGB. Это позволяет новой надписи визуально не отличаться от старой.
Замазка (Redaction): Исходный текст закрашивается белым прямоугольником поверх существующего контента.
Вставка нового текста: На место старого штампа вставляется новое значение (NEW_TEXT) через API рисования фигур (shape.insert_text). Текст принудительно выравнивается строго горизонтально (параметр rotate=0), что предотвращает случайный поворот символов на 90 градусов при обработке повернутых страниц.
Шрифты: Если указан путь к файлу .ttf, используется именно он; иначе применяется стандартный шрифт документа.
Сохранение: Обработанный файл сохраняется под новым именем с добавлением суффикса (по умолчанию _r), чтобы не перезаписать оригинал.
Структура проекта:

find_font_info: Вспомогательная функция, которая сканирует блоки страницы и возвращает параметры шрифта (размер и цвет) для конкретного прямоугольника.
replace_text_in_pdf: Основная функция редактирования. Принимает входной и выходной пути, списки текстов для замены и настройки шрифта. Возвращает общее количество произведенных замен.
find_pdfs: Утилита для сбора файлов в папке. Автоматически исключает из обработки документы, которые уже являются результатами (имеют заданный суффикс в названии).
Настройка переменных (в блоке if __name__ == "__main__"):

folder: Путь к директории с документами.
OLD_TEXTS: Список искомых строк (например, заглушек вида "XX_X").
NEW_TEXT: Значение, которое будет вписано вместо старых данных.
SUFFIX: Суффикс имени выходного файла.
FONT_PATH: Путь к кастомному TTF-шрифту (опционально).
