# Extracting a record from a drawing

Программа автоматически находит **шифр чертежа** в PDF‑файлах и **переименовывает** эти файлы по найденному шифру.

---

## Зачем это нужно?

В технической документации каждый чертёж имеет уникальный идентификатор — **шифр** (например, `GLE-RD-(P0-97-040)-N-LB 2600.001`).  
Часто файлы называются бессмысленно: `drawing1.pdf`, `чертёж_12.pdf` и т.п.  

Программа:

- сканирует все PDF в указанной папке,
- находит шифр на любой странице,
- **создаёт копию** каждого файла с именем, равным этому шифру.

В результате вы получаете папку, где все чертежи названы правильно и легко ищутся.

---

## Как это работает (очень просто)

Программа ищет шифр в два этапа:

1. **Быстрый способ** – читает текстовый слой PDF (как обычный текст).  
   Если шифр там есть – берёт его.

2. **Медленный, но надёжный** – если текстового слоя нет (или шифр не найден), программа:
   - ищет на странице якорь `GLE-RD-` (начало шифра),
   - вырезает область вокруг него,
   - отправляет этот кусочек в **OCR** (распознавание текста по картинке),
   - из распознанного текста вытаскивает полный шифр.

Это позволяет находить шифры даже на отсканированных чертежах, где нет текстового слоя.
